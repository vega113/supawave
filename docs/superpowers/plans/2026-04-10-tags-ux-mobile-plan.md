# Tags UX Mobile Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make tag chips filter search on tap, move removal to a separate mobile-friendly affordance, remove the blocking confirm popup, and document why model-level soft delete is not appropriate.

**Architecture:** Keep the search backend unchanged and add a small client event seam between the wave-panel tags UI and `SearchPresenter`. Redesign the tag markup/CSS so the chip body and remove affordance are distinct elements, then wire remove to a non-blocking toast-based flow.

**Tech Stack:** GWT client UI, Wave panel DOM/view builders, client event bus, JUnit tests, local browser verification.

- [ ] **Step 3: Add the new client event classes and registry hook**

Create the new event and handler modeled after `WaveSelectionEvent`:

```java
public class SearchQueryEvent extends GwtEvent<SearchQueryEventHandler> {
  public static final GwtEvent.Type<SearchQueryEventHandler> TYPE =
      new GwtEvent.Type<SearchQueryEventHandler>();

  private final String query;

  public SearchQueryEvent(String query) {
    this.query = Preconditions.checkNotNull(query, "null query");
  }

  public String getQuery() {
    return query;
  }

  @Override
  public Type<SearchQueryEventHandler> getAssociatedType() {
    return TYPE;
  }

  @Override
  protected void dispatch(SearchQueryEventHandler handler) {
    handler.onSearchQuery(query);
  }
}
```

```java
public abstract class SearchQueryEventHandler implements EventHandler {
  public abstract void onSearchQuery(String query);
}
```

Add to `ClientEvents`:

```java
  public HandlerRegistration addSearchQueryEventHandler(
      SearchQueryEventHandler handler) {
    return handlerManager.addHandler(SearchQueryEvent.TYPE, handler);
  }
```

```java
  private void handleTagFilterClicked(Element context) {
    TagView tagView = views.asTag(context);
    if (TagState.REMOVED.equals(tagView.getState())) {
      return;
    }
    Pair<Conversation, String> tag = models.getTag(tagView);
    if (tag != null) {
      ClientEvents.get().fireEvent(new SearchQueryEvent("tag:" + tag.second));
    }
  }
```

- [ ] **Step 4: Implement explicit remove without confirm popup**

In `handleTagRemoveClicked(...)`:

```java
  private void handleTagRemoveClicked(Element context) {
    Element tagElement = context.getParentElement();
    TagView tagView = views.asTag(tagElement);
    if (TagState.REMOVED.equals(tagView.getState())) {
      return;
    }
    Pair<Conversation, String> tag = models.getTag(tagView);
    if (tag != null) {
      tag.first.removeTag(tag.second);
      ToastNotification.showInfo("Removed tag: " + tag.second);
    }
  }
```
