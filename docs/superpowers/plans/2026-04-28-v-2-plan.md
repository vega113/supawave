## 1. Goal

`?view=j2cl-root` for a default signed-in user must not render any of:

- `Read.` — emitted by `J2clSelectedWaveModel.formatUnreadText` into `.sidecar-selected-unread`.
- `Live updates connected.` / `Live updates reconnected.` — emitted by `J2clSelectedWaveProjector.buildStatusText` into `.sidecar-selected-status`.
- The channel/snapshot URL line `wavelet · channel chN · snapshot vNNN` — emitted by `J2clSelectedWaveProjector.buildDetailText` into `.sidecar-selected-detail`.
- `Opened wave` — emitted by `J2clSelectedWaveView.mountColdCard` and by `HtmlRenderer.appendJ2clRootShellSelectedSnapshot` (and the preview-fixture renderer) into `.sidecar-eyebrow`.
- `Reply target: b+<id>` — emitted by `wavy-composer.js` (`<p class="target">`) and `composer-inline-reply.js`.

These data points stay accessible to developers under a new experimental flag `j2cl-debug-overlay` (default off in prod). With the flag on, every string above renders again in the same DOM location (no extra overlay UI required for this slice — the existing elements simply un-hide).

The J-UI version-history "Recent / Next unread / Previous / Next / End / Archive / Unpin" tab strip mentioned in the V-2 roadmap entry is a redesign owned by V-1 (rebuilt nav row); V-2 only suppresses dev strings.

## 2. Approach

Hybrid gating — CSS-hide for plain-DOM elements, render-gate for shadow-DOM Lit elements:

1. Add `j2cl-debug-overlay` to `KnownFeatureFlags` (default off, default-empty allowlist).
2. `WaveClientServlet` resolves the flag per viewer and forwards it to a new `renderJ2clRootShellPage` overload.
3. `HtmlRenderer.renderJ2clRootShellPage` adds CSS class `j2cl-debug-overlay-on` to `<body>` when the flag is on. The body already carries `j2cl-root-shell-page`, so the new class composes cleanly. The class is emitted before any sidecar content, and `sidecar.css` is `<head>`-loaded, so the hide rule applies before first paint (no FOUC).
4. **Plain-DOM (no shadow root) — hide via CSS**. Add to `j2cl/src/main/webapp/assets/sidecar.css`:
   ```
   body.j2cl-root-shell-page:not(.j2cl-debug-overlay-on) [data-j2cl-debug-only] { display: none !important; }
   ```
   Mark dev-string elements with `data-j2cl-debug-only="true"`:
   - `HtmlRenderer.appendJ2clRootShellSelectedSnapshot` (~line 4621): eyebrow, status, detail.
   - `HtmlRenderer.renderJ2clRootShellPage` preview-fixture block (~line 3866): eyebrow.
   - `J2clSelectedWaveView.mountColdCard`: eyebrow, status, detail (via `setAttribute`).
5. **`sidecar-selected-unread` `Read.` text**. Don't tag the element with `data-j2cl-debug-only` (would also hide the legitimate "X unread" count). Instead, in `J2clSelectedWaveView.applyState` and the equivalent server-first path, stop writing `Read.` to the DOM: when `model.isRead()` (or `getUnreadText().equals("Read.")` per existing model contract — both equivalent), set the element's `textContent` to empty and `hidden` attribute to `true`. The model's `getUnreadText()` keeps returning `"Read."` for unchanged tests and controller consumers. This avoids hardcoding English copy in a CSS gate.
6. **Lit elements with shadow DOM — gate at render time**. The global CSS rule cannot cross a shadow boundary. `wavy-composer.js` and `composer-inline-reply.js` are open-shadow `LitElement`. Add a `debugOverlay` Boolean property to each, defaulting to `false`. Read the body class once in `connectedCallback`:
   ```js
   this.debugOverlay = document.body?.classList.contains("j2cl-debug-overlay-on") ?? false;
   ```
   In `render()`, only emit `<p class="target">` when `this.debugOverlay`. The flag value is stable per page load, so reading once at connect is sufficient. Tests can set the property directly to exercise both branches without touching `document.body`.

This keeps models, projectors, and controller signatures untouched (`getUnreadText`, `getStatusText`, `buildDetailText` stay) — the visual change is confined to (a) DOM visibility for the SSR/Java view layer and (b) template branch in two Lit elements.

## 4. Feature-flag plumbing

Bridge overloads on `renderJ2clRootShellPage` keep callers compiling. Pattern matches the existing J-UI-1, J-UI-5, J-UI-8 flag plumbing. New overload signature appended at the end of the existing 12-arg form:

```java
public static String renderJ2clRootShellPage(
    JSONObject sessionJson, String analyticsAccount,
    String buildCommit, long serverBuildTime, String currentReleaseId,
    String rootShellReturnTarget, String websocketAddress,
    J2clSelectedWaveSnapshotRenderer.SnapshotResult snapshotResult,
    boolean railCardsEnabled,
    String viewerLocale,
    boolean serverFirstPaintEnabled,
    boolean inlineRichComposerEnabled,
    boolean debugOverlayEnabled);
```

Existing overloads delegate with `debugOverlayEnabled=false`. This is the fifth such overload — file a follow-up to refactor to a `J2clRootShellPageOptions` parameter object after V-2 lands. Out of scope here to keep the diff focused.

## 6. Local-server QA (gate)

1. From this worktree: `cd wave && sbt compile` then start dev server on a free port (default 9900): `bash scripts/worktree-boot.sh --port 9900`.
2. Register a fresh user via the sign-up flow (do not assume `vega` exists).
3. Sign in and navigate to `?view=j2cl-root`. Open a wave so the selected-wave card is mounted.
4. Capture **before** screenshot from `origin/main` HEAD: switch to a separate scratch worktree pinned to main on a different port; capture; save under `docs/superpowers/screenshots/v-2/before.png`.
5. Switch back to this worktree, restart server; capture **after** screenshot at the same window size and same wave selection; save `docs/superpowers/screenshots/v-2/after.png`.
6. Verify the after screenshot has zero occurrences of `Read.`, `Live updates connected.`, `Live updates reconnected.`, the `· channel chN · snapshot vNNN` line, `Opened wave`, and `Reply target: b+`.
7. Trigger a transport drop (kill the websocket server briefly, restore) so `Live updates reconnected.` would normally surface; confirm it still does not.
8. Toggle the flag on (`scripts/feature-flag.sh --local set j2cl-debug-overlay "..." --global && scripts/feature-flag.sh --local enable j2cl-debug-overlay`), reload, capture `docs/superpowers/screenshots/v-2/after-debug-on.png` showing the strings restored.
9. Edge cases:
   - Empty selected-wave state (no wave open) — eyebrow/status/detail must remain hidden.
   - Reply target paragraph: open reply composer; verify hidden when flag off, visible when on. Test both legacy `composer-inline-reply` (j2cl-inline-rich-composer flag off) and new `wavy-composer` (flag on).
   - Signed-out users on `?view=j2cl-root` — confirm none of the dev strings appear there either (sign-out path uses a different SSR branch; verify it stays clean).
   - `?view=gwt` path is unaffected — capture quick screenshot to confirm no regressions there (no body class change because `WaveClientServlet` only adds the new flag plumbing on the J2CL branch).
10. Capture **side-by-side annotation** for the PR body: numbered list mapping each removed string to its location in the before screenshot and the corresponding empty location in the after screenshot.

Dark mode and narrow-viewport rendering are explicitly out-of-sprint per the roadmap §6 (`07-dark-mode-shell.svg` and `06-mobile-inbox-and-wave.svg` are not in this sprint).

- Flag entry: `j2cl-debug-overlay` registered in `KnownFeatureFlags` (default off, no allowlist).
- Changelog fragment: `wave/config/changelog.d/2026-04-28-issue-1100-v-2.json`, summary noting that dev-only strings on `?view=j2cl-root` are now hidden by default and gated behind `j2cl-debug-overlay`.

## 8. Rollback

Disabling the slice is two lines: remove the `<body>` class branch and the `data-j2cl-debug-only` rule. With the flag on globally, behavior matches `main` exactly.
